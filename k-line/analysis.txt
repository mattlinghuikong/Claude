二、策略迭代 v1–v8 详解
================================================================================

【v1 — 朴素 K 均突破（K1 原书默认）】
─────────────────────────────────────────────
- 双均线 MA30 + MA150
- 入场：MA30 > MA150 + close 突破前 20 根 swing high
- 出场：close < MA150 OR close < swing low（OR 条件，容易被甩）
- 无止损、无加仓、无成本/趋势过滤

【v2 — ATR + 斜率 + 拖尾 + 加仓】
─────────────────────────────────────────────
A. ATR 突破缓冲：close 必须超过颈线 0.25×ATR
B. MA150 斜率过滤
C. ATR 拖尾止损（2.5×ATR）+ AND 出场条件
D. 单次顺势加仓

【v3 — v2 + 分市场参数 + ADX + 初始止损 + 冷却 + 量能】
─────────────────────────────────────────────
- ADX(14) 过滤：US > 15 / HK > 25 / CN > 28
- 初始固定止损 1.2-2.2×ATR，浮盈后切换拖尾
- MARKET_PARAMS 字典分市场调参
- 连亏冷却（HK/CN）+ 量能确认 + 分析师筛选

【v4 — v3 + 多周期共振 + 市场宏观 master switch + R-multiple】
─────────────────────────────────────────────
- 多周期共振：日线 + 周线 MA10/MA30
- 市场宏观 master switch（指数 ADX > 18）
- R-multiple 风险调整收益（每笔记录 stop_dist）

【v5 — v4 + 自动三级别 + RSI 均值回归备援 + 730d 小时线 mode】
─────────────────────────────────────────────
- 自动三级别（MA30+MA150+MA750）
- RSI 均值回归（震荡期备援）
- hourly_long 数据模式（730 天 1 小时）

【v6 — v4 + 配置化指标共振（BB/MACD/RSI）】
─────────────────────────────────────────────
- 提供 8 种过滤器组合的网格搜索
- 'bb'（中轨上方）/ 'macd'（柱>0）/ 'rsi'（zone）
- 发现：BB 在 trend strategy 下完全冗余

【v7 — v6 + 10% 硬止损 + BB %B + OBV + ATR 扩张】
─────────────────────────────────────────────
A. 10% 硬止损（绝对回撤上限）
B. BB %B 替代中轨（实证仍冗余）
C. NEW: OBV（量价双确认，独立维度）⭐
D. NEW: ATR 扩张过滤

- OBV 是 v7 最大单指标增益（美股 PF 1.92→2.11）
- ATR 扩张救港股 / A 股

【v8 — v7 删除 BB + 加 MFI / CCI / KDJ + CN 大熊期 master switch】
─────────────────────────────────────────────
A. 删除 BB（确认冗余）
B. NEW: MFI（量能 RSI）⭐ — 美股 PF 2.11→2.44
   - typical_price = (h+l+c)/3, money_flow = tp × volume
   - 长用 [50, 80] zone（买盘强但未过热）
   - 独立性：OBV 和 RSI 的混合，更稳

C. NEW: CCI（典型价偏离）
   - CCI = (tp - SMA(tp)) / (0.015 × MAD)

   - 独立性：用 typical_price 不只 close

D. NEW: KDJ（中文市场常用）⭐ — 港股 / A 股关键
   - K, D, J 三线
   - 长用 K > 50 + K > D + J > D

E. NEW: CN 大熊期 master switch
   - 仅对 A 股生效
   - 要求 CSI300 > MA200（多）/ < MA200（空）

三、关键发现
================================================================================

[1] 单笔胜率始终 33–37%，盈亏比是真正的赚钱来源
─────────────────────────────────────────────
所有变体中胜率从未突破 41%，但盈亏比从 v1 的 1.6 提升到 v8 的 2.6+。
书中"6 成胜率"目标在 2 级别 / 6 个月 / 3 年数据上不可达，但 PF > 1.4
就足以稳定盈利。


[2] ⚠️ 分析师纯打分筛选导致灾难性亏损（−490%）
─────────────────────────────────────────────
v3 随机选股 +505% → v3 + 分析师纯打分筛选 −490%
落差近 1000 个百分点。
根本原因：分析师"目标价 / 升值空间"打分本质是【均值回归预期】，
跟趋势跟随策略相位完全相反。


[3] BB 在趋势策略下完全冗余（多次验证）
─────────────────────────────────────────────
v6（BB > 中轨）→ 与 base 完全相同
v7（BB %B > 0.7）→ 与不带 BB 完全相同
v8 直接删除 BB

结构性原因：
- 现有策略已要求 MA30 > MA150 + close > 颈线 + 0.25×ATR
- 这些条件几乎必然让 close 在 BB 上半区
- BB 只是把 MA stack 已经表达的信息换个角度再说一遍

**结论：纯趋势策略下，BB 可以从工具箱里删除**


[4] OBV 是真正的新发现（v7 最大单指标增益）
─────────────────────────────────────────────
美股 v6_macd: PF 1.92, net +482%
美股 v7_obv: PF 2.06, net +502%（OBV 单独）
美股 v7_macd_obv: PF 2.11, net +513%（OBV + MACD）

为什么 OBV 独立？
- MA 看价格趋势，MACD 看价格趋势的快慢比
- OBV 看"上涨日的成交量减下跌日的成交量"
- 即使价格 MA 转好，如果 OBV 不跟，说明上涨"没有量"
- **量价双确认是独立信号维度**

OBV 在港股 / A 股没有显著增益（成交量数据噪声 + 散户主导）。


[5] MFI 是 v8 最大新发现（PF 2.11 → 2.44）
─────────────────────────────────────────────
美股 v7_macd_obv: PF 2.11, net +513%, 94 笔
美股 v8_macd_obv_mfi_atr_exp_cci: PF 2.44, net +300%, 40 笔

为什么 MFI 独立？
- OBV 只看方向（上涨日 vs 下跌日的量净累积）
- RSI 只看价格幅度（超买超卖）
- MFI = 量价加权的 RSI 双层验证
- 当 OBV、RSI 都通过但 MFI 不通过，往往是"上涨乏力的伪突破"

MFI 让交易笔数从 94→40（-57%），但 PF 从 2.11→2.44，
**单笔均收益从 +5.46% 提升到 +7.50%（+37%）**。

这是"少而精"路线 — 同样 60 只票上交易更挑剔，每次更值钱。


[6] KDJ 救港股 / A 股
─────────────────────────────────────────────
v7 港股 atr_exp: PF 1.09, net +15%
v8 港股 atr_exp + kdj: PF 1.13, net +22%

v7 A 股 atr_exp: PF 0.55, net -24%
v8 A 股 atr_exp + kdj + mfi: PF 0.68, net -12%（亏损减半）

KDJ 在港股 / A 股有用、在美股不显著的原因：
- KDJ 是 stochastic 的中文流派，对"区间内位置"特别敏感
- 港股 / A 股震荡时间多，KDJ 能识别短期超买超卖
- 美股趋势主导，KDJ 经常在过热区给假信号


[7] CN 大熊期 master switch 实证有效
─────────────────────────────────────────────
加了 CSI300 > MA200 强制条件后：
- A 股 v7（无）：PF 0.55, 净 −24%, 9 笔
- A 股 v8（有）：PF 0.68, 净 −12%, 6 笔

CSI300 跌破 MA200 期间被强制平仓不交易，避免了大熊期被反复磨损。
**这是 A 股第一个真正起作用的过滤器**（之前所有变体都亏）。


[8] ⚠️ 指标过滤 vs 票池筛选 — 二者不能简单叠加
─────────────────────────────────────────────
                  无 screen      + screen
─────────────────────────────────────────────
v4 (无指标过滤)     +472%         +677%   ← 票池筛选放大 +205%
v8 (全指标过滤)     +411%         +486%   ← 票池筛选放大 +75%
                    -61           -191
                                  ↑
                    指标过滤"挤掉"了票池筛选的增益

为什么？
- 票池筛选选出的是已经在涨的强势票 → MFI/KDJ/CCI 自然达标
- 指标过滤再加一层 → 等于只选"强势票里的最强"
- 结果：过滤器对"已经过滤过的票"无信号差，但减少了交易笔数
- 实质效果是票数减少 + 单笔质量微增，不如让票池筛选独自发力

**最优实战策略**：
- 重票池筛选 → 用 v4 + screen（最高净盈亏 +677%）
- 重风险控制 → 用 v8 + filters（不开 screen，PF 最高 2.44）


[9] RSI 在不同周期上效果相反
─────────────────────────────────────────────
- 日线：RSI 过滤损害收益（+472% → +231%）
- 小时线：RSI 过滤大幅改善（−44% → +45%）

这就是"周期决定指标用法"的实证 — 同一指标在不同时间尺度上的统计意义不同。


[10] 优化的边际收益顺序（最重要的认知）
─────────────────────────────────────────────
策略调参（v1 → v3）:           +850 个百分点（最大权重）
v4 多周期+master switch:       v4 vs v3 略减 daily，救小时线 +400+
v6/v7/v8 多指标过滤:           +25~+40 daily PF 改进
v8 CN master switch:           A 股 -24% → -12%（亏损减半）
正确的票池筛选:                +175~+450 daily 个百分点
错的票池筛选:                  -995 daily 个百分点（毁灭性）
**指标过滤 + 票池筛选叠加效应**: 互相挤压，1+1 < 2

→ 结论：先选战场（市场 / 票池 / 周期），再选战术（策略），最后调指标。


[11] R-multiple — 风险调整收益的硬指标
─────────────────────────────────────────────
v8 daily + 筛选:
  美股：120 笔，+60.9 R，单笔均 +0.51 R
  港股： 38 笔，+42.8 R，单笔均 +1.13 R
  A股 ： 13 笔，-15.3 R，单笔均 -1.18 R
  合计：171 笔，+88.5 R，单笔均 +0.52 R

v8 单笔均 R 与 v7 几乎相同（+0.51 vs +0.52），
说明在 screen 配置下两者表现等价 — 票池筛选已经把信号"用尽"。

九、Cybersecurity 与性能要点
================================================================================

【Cybersecurity 防护】
─────────────────────────────────────────────
1. 所有 ticker 入网前走严格正则白名单
   - YF_TICKER_RE = ^\^?[A-Z0-9.\-]{1,15}$  （允许 ^ 前缀给指数）
   - A_SHARE_RE   = ^\d{6}$
2. 缓存文件名只允许 [A-Z0-9._-]，并用 .resolve() 校验路径不能逃出
   data_cache/ 目录
3. 网络超时 30s；ThreadPoolExecutor 并发上限 5
4. 不用 pickle，缓存用 CSV / JSON（无反序列化攻击面）
5. 不用 eval / exec / subprocess 处理外部数据
6. HTTPS + 默认 SSL 校验，从不禁用证书验证
7. SCREENER_VERSION 缓存版本控制（当前 v4）
8. _strip_tz() 统一时区处理

【性能优化】
─────────────────────────────────────────────
1. CSV 磁盘缓存：重跑命中即不联网
2. ThreadPoolExecutor 平行 fetch（5 worker）
3. 滚动统计全用 pandas C 实现 (rolling, ewm)
4. 策略主循环跑 numpy 数组（省 ~20× 比 pandas Series 索引）
5. 早退条件：bar 数不够 MA150 + warmup 直接 skip
6. ATR / ADX / RSI / MACD / OBV / BB / MFI / CCI / KDJ / slope
   一次性向量化计算
7. 市场指数 regime 缓存到 _REGIME_CACHE，每市场只算 1 次
8. CN master switch 同样缓存
9. 周线 MA 用 pd.merge_asof 批量对齐，无逐 bar 查找
10. v8 网格 128 组合 × 60 票约 90 秒（含 60 票首次拉数据）
    缓存命中后约 50 秒（每组合 ~0.4 秒）
