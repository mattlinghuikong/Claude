# K均交易法 — 程序化交易实施备忘

来源：本目录下 `K1.docx` 与 `K2.docx`（作者：笑傲茶湖段）。两本书均为 100% 图片扫描页（K1 共 591 页 / K2 共 1584 页），无可提取文本，本文为视觉抽样阅读后的核心策略与注意事项的整合。

## 一、方法论总览

K 均交易法只看两样东西：**K 线** 和 **均线**。一切其它指标（MACD、KDJ、基本面、消息面、价值评估）都不参考，因为它们落后于价格本身——价格变了别人才发分析报告。

作者把交易系统分为三代：

| 代次 | 看什么 | 适用场景 |
|------|--------|----------|
| 一代系统 | 只看 K 线（高低点 + 颈线突破） | 入门，30 分钟及以上周期 |
| 二代系统 | 只看均线（小均线对比大均线） | 走势判定 |
| 三代系统 | K 线 + 均线相互印证 | 作者主力，赚钱最多 |

**核心交易哲学**

- **天道无常，按分类概率办事**：单笔不必赢，长期靠概率优势。作者声称 6 成胜率即可稳定盈利，70% 的单子是大赚，其余是小亏小赢。
- **机会出现就进场，机会消失就离场**：不预测，只识别。
- **等待是主业**：等待自己交易系统发出信号，不是等灵感。在价格高低点出现以前，永远无法知道，这就是交易的真谛。
- **进出点是一个区域，不是一个点位**：不要死扣某个具体价格。

## 二、价格波动规律（道氏理论的可执行翻译）

K 均交易法的高低点判定完全建立在道氏理论的三种走势定义之上：

- **上涨**：上升浪能连续不断地突破前期的高点，随后的回撤的低点能在前期的低点之上结束。简记：**新高之后是不新低**。
- **下跌**：反之。简记：**新低之前是不新高**。
- **盘整**：既不创新高也不创新低，或者既创新高又创新低（震荡假突破）。

派生规则：

- **创新高 / 创新低 → 趋势控制力依然存在**。
- **不创新低 + 新高 → 做多；反之做空**。这是最简单也最重要的两组排列关系。
- **小均线 > 大均线 → 大概率上涨**；反之大概率下跌。均线代表市场平均成本，K 线对均线的排列即趋势分类。

## 三、5 步交易系统（K1 第三章核心）

逐条都需要在程序里固化下来：

| 步骤 | 内容 | 程序化要点 |
|------|------|-----------|
| ① 固定操作级别 | 选定一个固定的 K 线周期 | 不在多周期之间漂移，避免主观切换 |
| ② 固定操作线 | 选定一条主均线（默认 **MA150**） | 与该周期匹配，确定后不动 |
| ③ 会画高低点 | 按道氏定义滚动维护当前高低点 | 实现一个 swing-high / swing-low 检测器 |
| ④ 确认突破标准 | 用 **颈线**（前高 / 前低水平线）作为突破阈值 | 突破颈线 = 信号触发，区分真假突破 |
| ⑤ 严格执行 | **到位就动，不到位就不动** | 信号触发即下单，无信号绝不动作 |

**额外细节**

- **加仓只能加一次**，且必须在突破方向被进一步确认后；浮亏加仓属赌徒思维，禁止。
- 单笔风险控制好的前提下，"加仓只能加一次"对应程序里的 `pyramid_count <= 1`。
- 入场后让利润奔跑，亏损迅速截断（顶级原则：迅速止损）。

## 四、三级别联立（K2 的进阶）

K2 在 K1 之上加了**三条均线 + 三种周期**的多级别协同。

### 4.1 三均线模型

软件标注为 `MA组合(0, 30, 750, 150, 0, 0)`，即同时挂三条均线：

| 角色 | 默认参数 | 颜色（书中例图） | 含义 |
|------|---------|-------------------|------|
| 大线 | **MA750** | 紫色 | 长期趋势过滤，相当于"老板说话" |
| 操作线 | **MA150** | 绿色 | 主操作均线，决定多空方向（K1 同款） |
| 小线 | **MA30** | 黄色 | 进出场细化、止损位参考 |

任意级别均线都对应一组高低点：黄色横线 = MA30 前高，绿色横线 = MA150 前高，依此类推；这就是"不同级别的高低点同时存在"。

### 4.2 周期划分

- **大周期**：1 小时、2 小时（超过 2 小时为"超大周期"，不推荐）。
- **30 分钟**：分界线，不大不小。
- **小周期**：15 / 10 / 5 / 3 / 1 分钟以及秒级。作者偏好**小周期**。

K2 上中下三篇分别对应：

- **上篇 — 中长线**（用 1H / 2H K 线 + MA(30,150,750)）
- **中篇 — 短线**
- **下篇 — 日内**

### 4.3 联立级别

- **单级别**：只看一个周期的一组均线。
- **两级别联立**：用大周期定方向 + 小周期定入场。
- **三级别联立**：长 / 中 / 短三周期同向，胜率最高。

操作时第一件事是明确：**这一笔做的是单级别、两级别联立、还是三级别联立？** 不同级别有不同分类规则，分别执行即可，不要把多个级别混在一起做精细化纠结。

### 4.4 突破与双突破

- **突破**：K 线突破前高/前低颈线。
- **双突破**：K 线 + 均线（操作线）同时被突破，更可靠的信号。
- 出现明显突破（单根光头光脚或近似）即翻向，例如下跌中明显突破上沿 → 翻空 / 翻多视方向而定。

## 五、资金管理

- **固定手数法**：在一个固定考核周期内（如月、季）固定每次开仓手数。
  - 例：1 万元起步且每次 1 手。账户到 2 万 → 改为 2 手。账户到 4 万 → 4 手。线性放大，不复利豪赌。
  - 回撤阶段（账户从 4 万跌回 2 万）→ 立即降回 2 手。**亏损时优先降手数，保住交易资格**。
- **直线加速规划（保守 200%/年）**：5 万本金五年目标 160 万，10 万本金五年目标 320 万。这是作者自己给的"心理锚定"，不是承诺。
- 单笔风险必须在系统层面截死，禁止扛单不止损（"鳄鱼法则"）。

## 六、心法与执行（容易被忽视但决定盈亏）

1. **悟性 → 心态 → 执行 → 结果**：成功交易是 信息→思考判断→行动→盈亏 的闭环；其中信息错（看错图、听错消息）和判断错占绝大多数亏损来源。
2. **不要追求精确度，先有节奏**：节奏（趋势）大过精确度。无节奏，无精确度。
3. **拒绝交叉验证多套体系**：找到一套有效的，长期一致地执行下去。
4. **反复盘整、假突破、震荡是必经之苦**。出现盘整不要在中段乱开仓，等下一次突破出现再做。
5. **不要频繁换品种**：选一个做下去；好坏都是概率，频换会把概率优势抽掉。建议**周期级别地换品种**（数月以上）而非天级别。
6. **空仓是状态，不是失败**：没有信号时，"什么都不做"本身就是优秀的交易。
7. **分类穷尽**：把所有 K 线 + 均线的相对位置组合（"X 仪 X 象 X 卦"式分类）穷举一遍并对应规则。书中反复强调，能完全分类才能机械执行。

## 七、对程序化实现的注意事项

将上述策略落到代码时，建议固化以下要点：

1. **可配置参数集合**（默认值取自书中 K2 模型）：
   - `period`: 1h / 2h / 30m / 15m / 5m
   - `ma_big = 750`, `ma_main = 150`, `ma_small = 30`（参数三选一同时启用，对应单/二/三级别联立）
   - `swing_lookback`: 用于识别 swing-high / swing-low 的回看根数
   - `breakout_buffer`: 颈线突破的最小幅度（避免假突破，用 ATR 或固定 bp）
   - `stop_loss_atr`: 单笔止损以 ATR 倍数为单位
   - `pyramid_count <= 1`：每个方向最多加仓一次
   - `position_units`: 当前手数；按账户权益分档调整

2. **状态机设计**：
   - `STATE`: `FLAT` / `LONG` / `SHORT`
   - `TREND`: `UP` / `DOWN` / `RANGING`（按"是否创新高 / 是否不创新低"分类）
   - `SIGNAL`: `BREAKOUT_UP` / `BREAKOUT_DOWN` / `DOUBLE_BREAKOUT_UP` / `DOUBLE_BREAKOUT_DOWN` / `NONE`
   - 切换条件全部以 K 线收盘为准（避免盘中假突破触发），这是程序化最稳妥的离散化点。

3. **多级别一致性检查**：仅当大 / 中 / 小三周期 `TREND` 同向时下三级别联立单；否则降级为两级别或单级别仓位。

4. **回测与实盘必须用同一份高低点定义**：把"创新高 / 不创新低"的判定函数单独抽出来做单元测试，因为它是整个策略的真值源头。

5. **禁项白名单**（程序里硬拒）：
   - 不在 `RANGING` 状态盘整中段开仓（必须等下一次颈线突破）。
   - 不浮亏加仓。
   - 不跨品种快速切换（设置最小持有周期）。
   - 信号未到不开仓（允许长时间空仓）。

6. **执行前自检清单**（每次发单前打印）：
   - 当前操作级别是？操作线是哪条？
   - 颈线在哪（前高 / 前低数值）？
   - 是否真突破（收盘价 vs 颈线 + buffer）？
   - 是否多级别同向？
   - 止损位 / 头寸单位是否合规？
